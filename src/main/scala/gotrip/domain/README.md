# Domain Models

В этой папке находятся все case class, описывающие бизнес-сущности сервиса GoTrip.

## Соглашения

- Для каждого кейс-класса определён неявный `Read` (иногда и `Write`) для Doobie, который позволяет преобразовывать строку БД в объект.
- Перечисления (`sealed trait`) размещены в том же файле, где и основная модель (например, `Role` в `UserRole.scala`). Для каждого перечисления задан `Meta[Enum]` с маппингом на строку.

## Список моделей

- `User` — учётная запись пользователя.
- `UserRole` — роль (USER/ADMIN).
- `NotificationPreference` — настройка push-уведомлений (одна на пользователя).
- `Location` — локация (страна, город, аэропорт, отель и т.д.).
- `Provider` — поставщик услуг.
- `Trip` — поездка пользователя.
- `TripLocation` — связь поездки с локацией (маршрут).
- `Order` — заказ (билет, бронь отеля и пр.).
- `OrderFile` — загруженный файл (ссылка).
- `OrderStatusEvent` — событие изменения статуса.
- `AdditionalService` — дополнительная услуга (каталог).
- `Review` — отзыв.
- `Achievement` — ачивка (каталог).
- `UserAchievement` — полученная пользователем ачивка.
